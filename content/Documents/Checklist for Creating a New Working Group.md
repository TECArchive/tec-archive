**About**
This checklist includes the actions needed in creating a new Working Group (WG) for TEC. This serves as a guide on how to properly set-up a new WG. The steward of the WG must ensure the fulfillment of most, if not all, of the checklist.

---

**Primary**

- [ ] Write the Working Group’s manifesto. [Example](https://docs.google.com/document/d/1cqdURiW7Qv50J_Ha9FLr9IvdqnZGuhlsffL6O1ivHnA/edit#)
- [ ] Draft the Working Groups’s roadmap. Add to the latest roadmap on this [Miro board](https://miro.com/app/board/o9J_lbkGJHQ=/)).
- [ ] Let the Community Stewards know on the #stewards channel!
- [ ] Join the Stewards call to present the new WG (how can we help!)

**Secondary**

- [ ] Set up a welcome document.
- [ ] Set up an Agenda document for your weekly/bi-weekly meeting.
- [ ] Set a schedule for the recurring Working Group Meeting to the TEC calendar. Reach out to for scheduling. (Add the Agenda doc to the calendar entry!)
- [ ] Add the Working Group as Topic under "Working Groups" category in the forum ([https://forum.tecommons.org/c/working-groups/13](https://forum.tecommons.org/c/working-groups/13)), and make a post about the WG
- [ ] If applicable, add a relevant label to the [coordination repository](https://app.zenhub.com/workspaces/tec-coordination-workspace-5fad0d3fbbe4da0011c2f40d/board?filterLogic=any&repos=306706322) in Zenhub.
- [ ] Add the WG title and manifesto/agendas on  [gitbook](https://app.gitbook.com/[REDACTED]-engineering-commons/s/tec-handbook/) under “Working Groups”.
- [ ] Create a [Discord](https://discord.gg/FxXn5m7A9Y) channel under “Working Group” category
- [ ] Choose WG emoji for the channel (see the others as examples).
- [ ] In the discord channel, pin a message containing relevant information and details about the WG for easy onboarding
- [ ] Create a discord voice channel under “Working Group VC” category
- [ ] Find out how to record meetings, and make sure all WG recordings are available via pawa (bot) or normal recording to upload. If the steward isn’t capable of recording the meeting, they may reach out to/ ( /) to do the recording. Once the recording is available,/ will upload the meeting in the Youtube channel.
- [ ] When the time is right: Set up template slides for the WG in the [Community Call Updates slidedeck](https://docs.google.com/presentation/d/1msWeZyBOURMEMqF_WtsmYqaBnBCQz7oKkKLp2xxpwIE/edit) and update (add slides - weekly updates) weekly
- [ ]  Make banners for the WG call/media so that those graphics can be shared on Twitter/telegram whenever there's a call or something ([https://docs.google.com/presentation/d/1A5z5IRhLeVEUgNb-h_USCVFZlmJFp5sDFDUt4QVEZ9Y/edit#slide=id.gcc4dd34b71_0_64](https://docs.google.com/presentation/d/1A5z5IRhLeVEUgNb-h_USCVFZlmJFp5sDFDUt4QVEZ9Y/edit#slide=id.gcc4dd34b71_0_64))
- [ ] Maybe check in with  () or with regard to the new TEC Infographic module, to get that Working Group added to the TEC Website.
- [ ] Contact (), if the WG needs to be bridged to some other discord or telegram group
- [ ] If it's a special WG (involving the need for some moderation powers/something of the sort), contact one of the stewards to get a new role made for the WG. (Example: Instances of WG that might be similar to Gravity, SoftGov or transparency)
- [ ] Set up Call Reminder for the WG’s discord channel (if needed). These are weekly reminders for the Working Groups call, which announces this in the WG channel 30 mins prior to the call. Stewards may reach out to  about this.
- [ ] Do the Transparency Audit (after a few months). Stewards may reach out to/ regarding this matter.
